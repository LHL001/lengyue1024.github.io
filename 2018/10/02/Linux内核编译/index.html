Linux内核编译

发表于 2018-10-02

Linux发行版本很多，但是他们共同的特征就是使用了Linux内核。Linux内核是基于C语言编写的符合POSIX标准的类unix操作系统。内核是操作系统最基本的部分，也是核心部分。Linux内核于1991年由当时的芬兰大学生Linus Torvalds开发。目前linus仍然致力于内核的设计、开发和维护，目前通过https://www.kernel.org网站发布与提供Linux所有版本的内核以及补丁。在GitHub上也有一份Linux 内核的镜像文件，地址:https://github.com/torvalds/linux。不过Linus 本人根本不想从 GitHub 接受 PR，因为质量太烂。各位可以拿来看看，但是不要随便发issue。

内核简介：
内核指的是一个提供硬件抽象层、磁盘及文件系统控制、多任务等功能的系统软件。一个内核并不是一套完整的操作系统。内核负责管理系统的进程，内存，设备驱动程序，文件和网络系统，决定着系统的性能和稳定性。
当前系统的内核存放到/boot/目录，我们可以进入该目录查看。

uname -r可以查看当前系统的内核版本。

说明：
4.15.0-34-generic
4代表的是内核主版本号；15代表的是次版本号，为不稳定版本(偶数是稳定版本，奇数是过渡版本)
0代表的是末版本号，属于修订版本号(扩展版本号)，表示修改次数。
(这是最新的内核版本命名方式，为第三种命名方式)
34为建立(build)号，每个建立可以增加少量新的驱动程序或缺陷修复。
generic 为通用版本的意思。 “pp”，就是“pre－patch”的意思，是个内核的测试版本。
首先下载内核。
我是需要提醒你的是，你最好在虚拟机中做这个实验，而且最好给虚拟机开快照，以便你编译出错的时候还可以回退回来。不至于重新再次安装系统。
网站：https://mirrors.edge.kernel.org/pub/linux/kernel/
这里我选择https://mirrors.edge.kernel.org/pub/linux/kernel/v4.x/中的4.18.11最新稳定版。

其中每个版本的内核都有2种不同的压缩方式，还有一个扩展名为sign的文件用于内核验证。

编译内核，目前我已知的有4种方式。
x config 该方法适合新手使用。基于图形化，编译起来更加简单易操作。
这里将的是make menuconfig方式编译。这种是最常用的编译内核的方式。
首先搭建环境。
安装gcc。
sudo apt-get install gcc
bingyu@ubuntu:~$ sudo apt-get install ncurses-dev

sudo apt-get install make
安装运行库

sudo apt-get install libncurses*
在正式编译内核之前，我们首先必须配置需要包含哪些模块。实际上，有一些非常简单的方式来配置。使用一个命令，你能拷贝当前内核的配置文件，然后使用可靠的 menuconfig 命令来做任何必要的更改。使用如下命令来完成：

sudo make menuconfig
